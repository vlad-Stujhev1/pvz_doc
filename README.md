# 📦 PVZ Document Archive Bot

Веб-приложение для загрузки, архивации и отправки документов с пунктов выдачи заказов (ПВЗ) через Telegram.

> 🤖 Тестовый бот: [@pvz_archive_bot](https://t.me/pvz_archive_bot)

---

## 🎯 Назначение

Система заменяет ручное ведение актов и отчётности, позволяя операторам ПВЗ:

- 📸 **Загружать фото** подписанных актов с системным хранением
- 🏷 **Категоризировать документы** (приёмка, возврат, доставка, инвентаризация и др.)
- ✍️ **Закрывать смены** быстро и прозрачно
- 📤 **Автоматически отправлять** файлы в Telegram Cloud

**Результат**: все документы в одном месте, под контролем, без хаоса.

---

## 🚀 Быстрый старт

### Требования
- Node.js ≥ v16
- Telegram Bot Token ([@BotFather](https://t.me/BotFather))
- Chat ID для получения файлов

### Установка

```bash
# 1. Клонируйте репозиторий
git clone https://github.com/your-username/pvz-archive-bot.git
cd pvz-archive-bot

# 2. Настройте бэкенд
cd server
npm install
cp .env.example .env
# Отредактируйте .env: укажите BOT_TOKEN и CHAT_ID

# 3. Запустите сервер
node index.js

# 4. Откройте frontend
# Откройте файл index.html в браузере или через локальный сервер
```

### Переменные окружения (`server/.env`)

```env
BOT_TOKEN=ваш_токен_бота
CHAT_ID=ваш_chat_id
PORT=3000 # опционально
```

## 📡 API (Backend)

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `POST` | `/api/upload` | Загрузка файла + метаданных (тип документа, смена, комментарий) |
| `GET`  | `/api/health` | Проверка статуса сервера |

**Пример запроса:**
```json
POST /api/upload
Content-Type: multipart/form-data

{
  "file": <binary>,
  "type": "acceptance",
  "shift": "2024-01-15",
  "comment": "Акт приёмки №123"
}
```

## 📦 Сборка и деплой

### Подготовка архива
```bash
# Очистка
rm -rf server/uploads/*
rm -rf server/node_modules frontend/node_modules

# Создание архива
tar -czvf pvz-archive-bot.tar.gz \
  --exclude='node_modules' \
  --exclude='.git' \
  --exclude='*.log' \
  .
```

- Бот должен быть добавлен в целевой чат и иметь права на отправку файлов
